# Self-Improve Entry: GitHub API 限流兜底（Repo Research）

- 日期：2026-02-16
- 触发来源：TASK-20260216-145106
- 风险等级：R1

## 问题

- 访问 GitHub REST API（`api.github.com`）在 unauthenticated 情况下容易触发 rate limit（HTTP 403, remaining=0），导致“仓库快速调研/介绍”流程中断或变慢。

## 改进方案

- 默认流程不依赖 GitHub REST API，优先使用：
  - raw githubusercontent 拉取 `README.md`/`LICENSE`/关键文档
  - `git clone --depth 1` 获取目录结构与关键入口
- 若确实需要 API 元数据（topics/default branch 等），优先用 `gh api`（OAuth）替代匿名 REST API。
- 在 Researcher 的 Skill Card/Playbook 中加入：
  - 限流信号（403 + rate limit headers）
  - 替代命令与注意事项（不粘贴 secrets）

## 验收标准

- 即使 GitHub REST API 被限流，仍可完成仓库概览输出，且 sources/skill card/记忆索引落盘完整。
- 任务日志明确记录是否触发限流以及采用的替代方案。
